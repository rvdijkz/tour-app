# Product Requirements Document (PRD)

# Overview

The application is a configurable cycling tour game platform supporting competitions such as:
- Tour de France
- Giro d’Italia
- Vuelta a España

The application allows players to create cycling teams based on configurable game rules and compete during a cycling tour based on stage results and scoring rules.

The system consists of:
- Web application
- Backend API
- Relational database
- Admin management interface
- Player interface

---

# Goals

## Primary Goals
- Allow players to create and manage a cycling team.
- Automatically calculate rankings and scores after each stage.
- Provide a transparent leaderboard with detailed score explanations.
- Allow administrators to manage riders, stage results, and game configuration.

## Secondary Goals
- Support configurable tours.
- Support configurable scoring rules and deadlines.
- Provide a responsive and easy-to-use GUI.

---

# User Roles

## Player
A participant in the game.

Capabilities:
- Login
- Change password
- Create and edit team before lock deadline
- View standings
- View rider list
- View detailed score breakdowns

Restrictions:
- Cannot modify teams after lock deadline
- Cannot manage riders or stage results

---

## Admin
Administrator of the game.

Capabilities:
- Create players
- Import riders via Excel
- Configure game settings
- Enter stage results
- Trigger score calculation
- Publish standings
- Manage tours and stages

---

# Functional Requirements

# Authentication

## Login
- Users authenticate with username and password.
- Initial passwords are generated by admin.
- Password reset/change required on first login.
- Initial password validity is configurable (default 12 hours).

## Authorization
Roles:
- PLAYER
- ADMIN

The application must support configurable tours:
- Tour de France
- Giro
- Vuelta
- Future tours

Configurable settings:
- Team lock deadline
- Double-point stages
- Maximum standard rider points
- Maximum reserve rider points
- Required nationality for special rider
- Initial password expiration duration

# Team Management

Each player owns exactly one team.

A team consists of 14 positions:

| Position | Type |
|---|---|
| 1-10 | Standard Riders |
| 11 | Nationality Rider |
| 12 | Reserve 1 |
| 13 | Reserve 2 |
| 14 | Kluns Rider |

Rules:
- Rider may only appear once in team
- Team modifications allowed only before lock deadline
- Point limits apply
- Kluns rider has no point limit

Additional team input:
- Predicted number of riders finishing the tour

---

# Team Validation Rules

## Standard Riders
Positions 1-11 combined:
- Maximum configurable point total

## Reserve Riders
Positions 12-13 combined:
- Separate configurable point total

## Nationality Rider
Position 11:
- Must match configured nationality

## Unique Riders
No duplicate riders allowed.

---

# Stage Result Management

Admin enters after each stage:
- Positions 1-20
- Last 5 riders
- Yellow jersey
- Green jersey
- Polka-dot jersey
- White jersey
- Most combative rider
- Abandoned riders

Stage must support:
- Double points configuration

---

# Scoring System

## Standard Stage Points

| Stage Position | Points |
|---|---|
| 1 | 12 |
| 2 | 10 |
| 3 | 8 |
| 4 | 7 |
| 5 | 6 |
| 6 | 5 |
| 7 | 4 |
| 8 | 3 |
| 9 | 2 |
| 10 | 1 |

---

## Team Position Bonus

For team positions 1-10:
- Exact match between rider stage position and team position:
  - +2 bonus points

---

## Nationality Rider Scoring

Position 11:
- Standard scoring applies for stage positions 1-8
- Stage positions 9-20 award 3 points

---

## Rider Abandonment

If riders at positions 1-11 abandon:
- -2 points penalty

Reserve riders:
- Replace abandoned riders from next stage onward
- Reserve 1 replaces highest abandoned position first
- Reserve 2 replaces next abandoned position

Reserve rider abandonment before substitution:
- No penalty
- Rider becomes unavailable

---

## Double Point Stages

If stage configured as double points:
- All points for positions 1-11 are doubled

---

## Jersey Points

Applicable for team positions 1-11.

| Jersey | Points |
|---|---|
| Yellow | 3 |
| White | 1 |
| Green | 1 |
| Polka Dot | 1 |

Jersey points become active starting next stage.

---

## Combativity

Applicable for positions 1-10:
- Most combative rider:
  - +5 points

---

## Kluns Rider Rules

| Result | Points |
|---|---|
| Last place | +5 |
| 2nd-5th last | +2 |
| Positions 11-20 | -5 |
| Positions 2-9 | -10 |
| Stage winner | -15 |
